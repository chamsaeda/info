🏠 IP 주소가 뭐예요?

IP 주소는 인터넷에 연결된 컴퓨터나 스마트폰의 주소예요. 마치 우리 집에 주소가 있어서 택배를 받을 수 있는 것처럼, 컴퓨터도 IP 주소가 있어야 인터넷에서 데이터를 주고받을 수 있어요!

📍 예시: 192.168.0.1 처럼 점(.)으로 구분된 숫자들이에요.

🌍 공인 IP와 사설 IP는 왜 필요할까요?

💡 왜 공인 IP와 사설 IP로 나눴을까요?
❌ 문제: IP 주소가 부족해요!

IPv4는 약 43억 개의 주소만 만들 수 있어요. 하지만 전 세계 인터넷 기기는 수백억 개! 모든 기기에 공인 IP를 주면 주소가 부족해요.

✅ 해결: 사설 IP를 사용해요!

집마다 하나의 공인 IP만 사용하고, 집 안 기기들은 사설 IP를 사용해요. 이렇게 하면 43억 개의 공인 IP로 전 세계 모든 사람이 인터넷을 쓸 수 있어요!

🔄 NAT가 뭔가요?

NAT는 Network Address Translation의 약자로, "네트워크 주소 변환"이라는 뜻이에요.

쉽게 말하면, 공유기가 사설 IP를 공인 IP로 바꿔주는 일을 해요! 마치 통역사처럼요! 🗣️

  • ✗포트 포워딩 필요: 서버 운영 시 추가 설정 필요

🏘️ 서브넷(Subnet)이 뭐예요?

인터넷 전체를 거대한 도시라고 하면, 서브넷은 그 도시 안의 아파트 단지예요. 하나의 큰 네트워크를 용도나 장소에 따라 작은 구역으로 나눈 것이 서브넷이에요.

🌆 인터넷 (거대한 도시)
 ├── 🏢 1반 네트워크  (192.168.1.???)
 │    ├── PC-01 (192.168.1.1)
 │    └── PC-02 (192.168.1.2)
 └── 🏢 2반 네트워크  (192.168.2.???)
     ├── PC-01 (192.168.2.1)
     └── PC-02 (192.168.2.2)
🤔 왜 굳이 나눌까요?
🚦
교통 정리 — 컴퓨터가 많을수록 신호가 뒤엉켜요. 반별로 나누면 각 반 안에서만 신호가 돌아서 훨씬 빠르고 조용해져요.
🔒
보안 — 서버실(2반)과 학생 PC(1반)를 다른 서브넷으로 분리하면, 학생이 실수로 서버에 접근하는 걸 막을 수 있어요.
📦
효율 — 필요한 만큼만 IP를 나눠쓰기 때문에 IP 주소 낭비가 없어요.
🚧 그럼 어떻게 접근을 막아요? — 라우터가 경비원!

서브넷을 나누는 것만으로는 자동으로 막히지 않아요. 서로 다른 서브넷 사이에는 반드시 라우터(Router)를 거쳐야 하는데, 바로 이 라우터에 "1반 → 2반 통행 금지" 규칙을 설정하면 차단돼요.

🖥️ 1반 PC (192.168.1.10)
      │ "2반 서버에 접속하고 싶어요!"
      ▼
  🚦 라우터 (교문 경비원)
      │ 규칙 확인 중... ❌ 1반은 2반 접근 불가!
      ✖ 차단!
🖥️ 2반 서버 (192.168.2.10) ← 도달 못 함

💡 같은 서브넷 안의 컴퓨터끼리는 라우터를 거치지 않고 직접 통신해요. 다른 서브넷으로 가려면 무조건 라우터를 통과해야 하기 때문에, 라우터가 유일한 출입구가 되는 거예요.

✂️ /숫자는 무슨 뜻이에요? (CIDR 표기법)

IP 주소 32비트 중에서 앞 몇 비트를 "동 번호(네트워크)"로 쓸지 결정해요. 숫자가 클수록 네트워크가 작아지고, 연결 가능한 기기 수도 줄어요.

🤔 라우터(Router)와 게이트웨이(Gateway)는 다른 건가요?

많이 헷갈리는 개념이에요! 한마디로 정리하면 — 게이트웨이는 역할이고, 라우터는 장치예요. 마치 "경비원"이 역할이고 "아저씨"가 그 일을 하는 사람인 것처럼요.

📡
라우터 (Router)
= 장치 (Hardware)
  • ▸네트워크와 네트워크 사이에서 데이터를 어느 길로 보낼지 결정하는 장치예요.
  • ▸집에 있는 공유기(Wi-Fi 기계)가 대표적인 라우터예요.
  • ▸여러 길(경로) 중에서 가장 빠른 길을 골라줘요.
🚪
게이트웨이 (Gateway)
= 역할 (Role)
  • ▸서로 다른 네트워크를 연결해주는 "출입구 역할"을 뜻하는 말이에요.
  • ▸라우터가 게이트웨이 역할을 하는 경우가 가장 많아요.
  • ▸라우터 외에도 방화벽, 스위치 등이 게이트웨이 역할을 할 수 있어요.
