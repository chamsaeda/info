🗜️ 무손실 압축 알고리즘

데이터를 줄여도 원래대로 복구할 수 있는 마법 같은 기술!

💡 무손실 압축이란?

원본 데이터를 더 작은 용량으로 만들되, 압축을 풀면 원본과 똑같이 복원되는 압축 방법이에요. 마치 접었던 종이를 펴면 원래대로 돌아오는 것처럼, 데이터를 손실 없이 압축했다가 완벽하게 복원할 수 있어요!

🔄 RLE (Run-Length Encoding)란?

RLE는 같은 데이터가 연속으로 반복될 때 "문자+개수"로 표현하는 압축 방법이에요. 예를 들어 "aaaaaabbbb"를 "a6b4"로 줄일 수 있어요! 6개의 a와 4개의 b라는 의미죠! 간단하지만 효과적인 압축 방법이에요!

💡 활용: 흑백 이미지, 팩스 문서, 단순한 그림 파일, 같은 값이 많이 반복되는 데이터 등

📝 텍스트 압축 체험

💡 반복이 없으면 RLE는 오히려 용량이 커져요!

🎨 이미지 압축 체험 (픽셀 클릭으로 그림 그리기)

행별 RLE 압축 결과:
💡 RLE 압축 규칙:
• 항상 흰색부터 시작하여 흰색과 검은색이 번갈아 나타난다고 가정합니다.
• 각 숫자는 해당 색상의 연속 개수를 의미합니다.
• 예: "0, 3, 1, 1" = 흰색 0개, 검은색 3개, 흰색 1개, 검은색 1개
• 예: "5" = 흰색 5개 (검은색 없음)

📌 RLE의 특징

👍 장점

  • • 알고리즘이 매우 간단해요.
  • • 압축/해제 속도가 빨라요.
  • • 반복이 많으면 압축률이 뛰어나요.

👎 단점

  • • 반복이 없으면 오히려 커져요.
  • • 다양한 색상에는 비효율적이에요.
  • • 복잡한 데이터에는 부적합해요.

🌳 허프만 코딩(Huffman Coding)이란?

허프만 코딩은 자주 나오는 문자에 짧은 코드를, 드물게 나오는 문자에 긴 코드를 부여하는 방법이에요. 마치 자주 쓰는 단어는 줄임말로 쓰는 것과 비슷해요! 예를 들어 "aaa...b"처럼 a가 많이 나오면 a에 "0", b에 "10"처럼 짧은 코드를 붙여서 용량을 줄여요.

💡 활용: ZIP 파일, JPEG 이미지의 일부, 텍스트 압축, 데이터를 빠르게 전송할 때 등

🔢 허프만 코딩 원리

1.각 문자의 빈도수를 계산합니다.
2.빈도수가 낮은 순서대로 정렬합니다.
3.가장 낮은 2개를 합쳐 노드를 만듭니다.
4.트리가 완성될 때까지 반복합니다.
5.왼쪽은 0, 오른쪽은 1을 부여합니다.

✨ 허프만 코딩 체험

💡 모든 문자의 빈도가 같으면 허프만 코드 길이가 균일해져 압축 효과가 거의 없어요!

📌 허프만 코딩의 특징

👍 장점

  • • 빈도 차이가 크면 압축률이 높아요.
  • • 최적의 접두 부호를 만들어요.
  • • 구현이 매우 단순하고 속도가 빨라요.

👎 단점

  • • 빈도가 비슷하면 효과가 적어요.
  • • 트리 정보도 함께 저장해야 해요.
  • • 2번 스캔이 필요해요. (빈도 계산 + 압축)

📖 사전 기반 압축(Dictionary-based Compression)이란?

사전 기반 압축은 자주 나오는 단어나 패턴을 사전에 등록하고, 짧은 기호로 바꾸는 방법이에요. 마치 "ㅋㅋ"을 웃음으로 약속하는 것처럼요!

💡 활용: 문서 파일, 프로그램 코드, HTML 웹페이지, 반복되는 문구가 많은 텍스트 등

✨ 사전 기반 압축 체험

📌 실제 사전 기반 압축(LZW 등)은 압축하면서 동시에 사전을 자동으로 만들어가요. 여기서는 먼저 반복되는 단어를 찾아서 짧은 기호로 바꾸는 방식으로 원리를 쉽게 체험할 수 있도록 단순하게 만들었어요.

📌 사전 기반 압축의 특징

👍 장점

  • • 반복되는 패턴에 효과적이에요
  • • 단어 단위 압축이 가능해요
  • • 문서 파일에 적합해요

👎 단점

  • • 사전도 함께 저장해야 해요
  • • 반복이 없으면 비효율적이에요
  • • 사전 크기에 따라 효율이 달라요

🔍 LZ77 압축(Lempel-Ziv Compression : 개발자 이름을 딴 압축법)이란?

LZ77은 이미 나왔던 데이터를 다시 가져오는 방법이에요. "7칸 앞으로 가서 4글자 가져와"처럼 (거리, 길이)로 표현합니다. 같은 내용이 반복될 때 "아까 그거랑 똑같아!"라고 표시만 해두는 거예요!

💡 활용: ZIP, PNG, GZIP 등 - 가장 많이 사용되는 압축 방식!

🪟 슬라이딩 윈도우 원리

1.현재 위치 앞쪽의 일정 범위를 검색합니다 (슬라이딩 윈도우)
2.같은 패턴이 있는지 찾아봅니다
3.발견하면 (거리, 길이)로 표현해요
4.못 찾으면 문자 그대로 출력해요

✨ LZ77 압축 체험

💡 반복 패턴이 없는 문자열은 LZ 압축 시 오히려 커질 수 있어요!

🔍 LZ 계열 압축 두 가지 구분하기

LZ77 (슬라이딩 윈도우)

→ 이미 나온 데이터를 뒤쪽에서 참조

→ (거리, 길이) 쌍으로 표현

📁 ZIP, PNG, GIF(일부), DEFLATE에서 사용

LZW (LZ78의 변형)

→ 압축하면서 사전을 동적으로 생성

→ 사전 번호(①, ②...)만 기록

📁 GIF, TIFF, PDF, 유닉스 compress에서 사용

💡 이 페이지의 LZ 압축 체험은 LZ77 방식을 기반으로 합니다. 요즘은 LZ77 계열(DEFLATE)이 ZIP, gzip, PNG, 웹 압축 등에서 압도적으로 많이 사용됩니다.

📌 LZ77의 특징

👍 장점

  • • 반복 패턴에 매우 효과적이에요.
  • • 사전 저장이 필요 없어요.
  • • 메모리 사용이 적어요.

👎 단점

  • • 검색 과정이 필요해요.
  • • 반복이 없으면 효과가 적어요.
  • • 윈도우 크기에 따라 성능이 달라요.

🤔 학생들이 자주 헷갈리는 3가지

1. RLE는 왜 색이 자주 바뀌면 용량이 오히려 커질까?

문자마다 숫자(길이)를 붙여야 하기 때문이에요. 예를 들어 "abcd" → "a1b1c1d1"로 두 배가 됩니다. 반복이 없을수록 불리해요!

2. 허프만 코딩은 왜 항상 최적이라고 할까?

주어진 문자 빈도에 대해 최적의 접두 부호(prefix code)를 만들어줍니다. 단, 빈도가 모두 같으면 코드 길이도 균일해져 압축 효과가 줄어들어요.

3. 허프만 트리는 항상 하나만 만들어질까?

아니요! 같은 빈도의 문자가 있을 때 트리는 여러 개 존재할 수 있어요. 어느 트리를 선택해도 총 비트 수는 동일하게 최적입니다.

🎯 어떤 압축이 더 효율적일까?

데이터 특성에 따라 더 잘 맞는 압축 방법이 있어요. 각 데이터에 어떤 압축이 더 유리할지 생각해보세요!

📄

데이터: AAAAAAA

같은 문자가 길게 반복돼요.

✅ 더 유리한 압축: RLE

연속 반복 패턴 → RLE가 "A7" 하나로 압축! 허프만도 쓸 수 있지만 이 경우엔 RLE가 가장 간단해요.

🔁

데이터: HelloHelloHello

같은 단어가 3번 반복돼요.

✅ 더 유리한 압축: LZ77 / 사전 기반

동일 패턴 반복 → LZ77은 (거리,길이)로, 사전 기반은 기호로 치환. 둘 다 효과적이에요!

🎲

데이터: 무작위 데이터 (랜덤)

패턴이나 반복이 전혀 없어요.

✅ 더 유리한 압축: 효과 없음

어떤 압축을 써도 효과가 없거나 오히려 커질 수 있어요. 이미 압축된 파일(ZIP, MP3)도 같은 이유로 재압축이 안 돼요!

🤔 생각해보기

Q. 무손실 압축과 손실 압축의 차이는?

무손실 압축은 원본을 100% 복원할 수 있어요. 문서, 프로그램처럼 정확해야 하는 파일에 사용합니다. 손실 압축은 일부 정보를 버려서 더 작게 만들어요. 사진, 음악, 동영상처럼 약간의 손실이 괜찮은 파일에 사용합니다.

Q. ZIP 파일은 어떤 압축을 사용하나요?

ZIP 파일은 주로 LZ 압축과 허프만 코딩을 함께 사용해요. 먼저 LZ로 반복 패턴을 찾고, 그 다음 허프만 코딩으로 빈도가 높은 데이터를 더 압축합니다!

Q. 압축률은 항상 좋을까요?

아니요! 반복이나 패턴이 적은 데이터는 압축률이 낮아요. 예를 들어 완전히 랜덤한 데이터나 이미 압축된 파일은 더 이상 압축이 안 될 수 있어요.

Q. 실생활에서 무손실 압축을 어디에 쓸까요?

문서 전송, 프로그램 설치파일, 의료 이미지, 법률 문서 등 원본이 중요한 모든 곳에서 사용됩니다!
